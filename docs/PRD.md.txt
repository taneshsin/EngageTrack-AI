# 📘 Product Requirements Document – EngageTrack AI

## 1. 🧭 Overview

EngageTrack AI is a simulated lifecycle insight and productivity enhancement platform. It is built to demonstrate AI-powered feature nudging, engagement scoring, churn detection, and A/B variant testing — without relying on sensitive user data.

---

## 2. 🎯 Goals

- Simulate a working productivity analytics platform for SaaS users
- Demonstrate how AI-powered nudges can improve user retention
- Show DevOps skills via containerization, CI/CD, and AKS hosting
- Make it recruiter-facing, deployable, and product manager ready

---

## 3. 👥 Target Users (Simulated Personas)

| Persona   | Characteristics                            |
|-----------|--------------------------------------------|
| Student   | Low attention span, likes gamified nudges  |
| Marketer  | Seeks campaign performance tips            |
| Analyst   | Focused on insights and dashboard clarity  |
| Writer    | Prefers quiet engagement reminders         |

---

## 4. 🧩 Key Features

| Feature                          | Description |
|----------------------------------|-------------|
| AI-Generated Nudges              | GPT-like text suggestions for user re-engagement |
| Engagement & Churn Dashboard     | High-level metrics and distribution views |
| A/B Variant Assignment           | Random assignment to simulate experiment setup |
| Per-user Detail View             | Drill-down by persona, churn risk, and feature use |
| Download Summary                 | One-click export of user lifecycle insights |
| Logging & Mock API               | Simulated analytics + user nudge generation logs |

---

## 5. 📈 Metrics for Success

| Metric                | Target |
|-----------------------|--------|
| Successful CI/CD run  | ✅     |
| User nudge generation | ✅     |
| App uptime (AKS)      | ✅     |
| Variant balance       | ~50/50 |
